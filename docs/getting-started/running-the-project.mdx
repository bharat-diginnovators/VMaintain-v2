
# ATS Project Documentation

## Overview

The ATS (Automated Tracking System) project is designed to maintain and monitor websites efficiently. Key features include:

- **Site Health Monitoring**: View the health status of various sites.
- **Progress Tracking**: Track and monitor the progress of different sites.
- **Client Reports**: Generate and view detailed reports for clients.
- **Fully Responsive Design**: Optimized for various screen sizes and devices.

The project is built using **React** and **Tailwind CSS**, providing a robust, responsive, and user-friendly interface. This documentation will guide you through the reusable components and modular code used in the project.

## Features

- **Reusable Components**: Components that can be used throughout the project for consistency and ease of maintenance.
- **Modular Code**: Code is structured in a modular fashion to enhance readability and maintainability.
- **Customizable**: Easily customize components and features as per project requirements.

## Installation

1. **Clone the Repository**

   ```bash
   git clone https://github.com/your-username/ats-project.git
   ```

2. **Navigate to the Project Directory**

   ```bash
   cd ats-project
   ```

3. **Install Dependencies**

   ```bash
   npm install
   ```

4. **Run the Project**

   ```bash
   npm start
   ```

## Components

### Profile Component

The Profile component includes form inputs for user information and allows users to upload a profile image.

**Key Features:**

- Form inputs for user data (e.g., first name, last name, email, address).
- Profile image upload functionality.

### Sidebar Component

The Sidebar component provides navigation and responsiveness.

**Key Features:**

- Opens or closes based on screen size (medium or larger).
- Active tabs reflect current navigation state.
- On large screens, the sidebar shows icons when closed and text when open.

### Data Table

The DataTable component displays site-related data with customizable columns.

**Key Features:**

- **Status Column**: Displays radio button icons with color indicators (green for active, gray for inactive).
- **Site URL Column**: Includes clickable links with icons.
- **Site Health Column**: Shows health status with 'Good,' 'Bad,' or 'Worst' indicators.

## Styling with Tailwind CSS

The project uses Tailwind CSS for styling, providing utility-first CSS classes to design components.

**Key Points:**

- Custom styles applied using Tailwind CSS utilities.
- Responsive design to ensure compatibility with various devices.

## State Management with Redux

Redux is used to manage the application state, especially for handling user profile images and data.

**Key Points:**

- Profile image upload and display functionality is managed through Redux.
- Scalable state management for handling various data interactions.
